Elevation indicator v2 — one row, still loud

v1 added a second tinted row below the action controls. The combined bar grew to ~110 px, clipped the Refresh / Cancel / Start build buttons in the live app, and competed with the existing "Inputs locked" hint inside the Setup card. v2 keeps the bottom bar to exactly one row by pouring the elevation cue into the existing action-bar rather than next to it.

Hard constraints

One row only. Active height == idle height. Refresh / Cancel / Start build stay on the right and are never displaced. Status pill + ProgressBar stay where they are. Phase label updates live (Validate / Mount / Wipe / Partition / Copy / Split / Verify).

What must still happen

When the elevated worker is running, the bar must be unmistakably louder than the idle "asInvoker" caption that the user previously called "so hidden". Theme-aware (WinUI semantic brushes); both light and dark must clear contrast at ≥4.5:1 on body text. Animations respect prefers-reduced-motion.

"i want the bottom as a single line. this is crappy." — @crutkas, on the v1 two-row band
 RECOMMENDED
Option A

Tinted action row + admin pill

No new row. The existing action-bar tints to soft accent, the status pill morphs into a loud accent-filled SHIELD · ADMIN · COPYING chip, and a warning-tinted drive chip sits between the progress bar and the buttons. Same height in both states.

Behavior

  • Idle → neutral pill ("Ready · click Start"), no tint, no drive chip.
  • Active → bar fills with accent tint, sheen sweeps L→R every 3.6 s.
  • Status pill becomes accent-filled with shield + phase name + pulse dot.
  • Progress bar uses a shimmering accent fill (vs static idle).
  • Drive chip uses the warn color to call out the destructive target.
  • Start build dims to 45% (still visible, clearly inactive); Cancel stays full strength.

Why it wins

  • Zero height change — solves the user's exact complaint.
  • Two redundant cues (chip + tint + animated fill) make it impossible to miss.
  • The phase label lives in the loudest element on the bar, so updates are obvious.
  • Drive callout preserves the most destructive bit of context that the v1 band carried.
  • Maps cleanly to WinUI brushes: AccentFillColorDefaultBrush + TextOnAccentFillColorPrimaryBrush for the pill; SubtleFillColorTertiaryBrush tinted via AccentAcrylicBackgroundFillColorBaseBrush for the row; SystemFillColorCautionBrush for the drive chip.

Option B

Slim accent rail + inline shield badge

The action-bar gets a chunky 3 px accent top border and a soft 5% accent wash. A square shield badge appears at the far left followed by uppercase "ADMIN ACCESS" and a phase chip. Quiet, refined, but louder than v0.

Behavior

  • Idle → no rail, neutral pill at the left.
  • Active → 3 px accent rail along the top of the bar.
  • Shield badge + caps label replace the existing status pill.
  • Phase chip uses the soft accent tint and updates live.
  • Progress and buttons unchanged in layout.

Trade-off

  • Most restrained option. Closest to current UI.
  • Risk: the user previously called the 11 pt caption "so hidden" — a 3 px rail alone might still feel timid. The square shield badge + caps lockup mitigate that.
  • Good fit if we want to feel premium and Fluent-quiet rather than alarming.

Option C

Side-flag lockup

A chunky 6 px accent bar runs along the left edge of the action row, followed by a square shield badge and an uppercase ADMIN ACCESS · COPYING lockup. Reads like a Visual Studio "Debugging" mode strip — high signal, low chrome.

Behavior

  • Idle → no flag, neutral pill.
  • Active → 6 px accent flag, soft accent wash, caps lockup, phase + pulse.
  • Status pill is replaced by the lockup; progress + buttons unchanged.
  • Reads strongly as a "mode" indicator, not just a status.

Trade-off

  • Loudest of the three without sacrificing the single-row rule.
  • The chunky left flag can feel a bit "debugger" / developer-UI in a consumer-friendly Fluent app.
  • Strong choice if the team wants the bar to look different in kind when elevated, not just tinted.

Recommended

Ship Option A — Tinted action row + admin pill

Option A directly solves the user's complaint ("i want the bottom as a single line. this is crappy.") without sacrificing the loud, can't-miss elevation cue that motivated v1 in the first place. The action-bar height stays pinned to ~34 px in both states; the buttons never move.

  • Loud: three redundant cues — accent-tinted row, accent-filled SHIELD · ADMIN · COPYING pill, and a shimmering progress fill — make the elevated state obvious from across the room.
  • Phase clarity: the live phase name (Validate / Mount / Wipe / Partition / Copy / Split / Verify) sits inside the loudest element on the screen, so users always know what the admin worker is doing right now.
  • Context preserved: the warn-tinted drive chip (Disk 1 · USB DISK 3.0 · 57.8 GB) carries the most destructive bit of context that v1's band carried — but inline, not in a second row.
  • WinUI-native: maps to AccentFillColorDefaultBrush / TextOnAccentFillColorPrimaryBrush for the pill, AccentControlElevationBorderBrush for the row border, SystemFillColorCautionBrush for the drive chip, and uses color-mix-style soft tints that already work with both light and dark themes.
  • Supersedes: with the bottom bar now loudly indicating elevation, the duplicate "Inputs locked. Cancel to make changes…" footer hint inside the Setup card can be downgraded to a one-line caption or removed entirely.

Fallback: if user testing finds Option A too "alarm-y" for a successful, expected admin operation, Option B is the safer Fluent-quiet variant that still adds a clear shield badge + caps label without flooding the whole row.
